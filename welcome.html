Apollo Kotlin Normalized Cache Help

Welcome

This is Apollo Kotlin's Normalized Cache.

For an introduction please read the Normalized Cache documentation.

The Normalized Cache is now hosted in this dedicated repository and published at its own cadence and versioning scheme.

Compared to the previous version, this library brings:

  • Pagination support

  • Expiration (a.k.a. Time to live)

  • Garbage collection, and trimming

  • Partial results from the cache

  • API simplifications

  • Key scope support

  • SQL cache improved performance

The Normalized Cache in the Apollo Kotlin main repository will not receive new features - they are added here instead. The main repository version is deprecated in Apollo Kotlin v5, and it will be removed in the future.

Use in your project

The Normalized Cache requires Apollo Kotlin v4.3.0 or later.

  1. Add the dependencies to your project

// build.gradle.kts dependencies { // For the memory cache implementation("com.apollographql.cache:normalized-cache:1.0.5") // For the SQL cache implementation("com.apollographql.cache:normalized-cache-sqlite:1.0.5") }

  1. Configure the compiler plugin

// build.gradle.kts apollo { service("service") { // ... // Add this plugin("com.apollographql.cache:normalized-cache-apollo-compiler-plugin:1.0.5") pluginArgument("com.apollographql.cache.packageName", packageName.get()) } }
// For Apollo Kotlin v4 apollo { service("service") { // ... // Add this plugin("com.apollographql.cache:normalized-cache-apollo-compiler-plugin:1.0.5") { argument("com.apollographql.cache.packageName", packageName.get()) } } }

If you're already using the main repository version, please consult the migration guide.
